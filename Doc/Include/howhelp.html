How to Help to the library

Development Help

Work on the library is more and more time consuming for me and I don't have enough time for it.

So, if you want to help with development please contact me at and the team will find an area of the library that you can maintain

You can help with any part that you like:

  • Core Development
  • Demo App Development
  • Wizard Development
  • Write some Examples
  • MONO port
  • Help Translation, it is almost to English :) (I know that is full of errors)
  • Web Site Changes
  • Forums Admin
  • Or, if you don't want to work, you are welcome to donate :P
If you change any part of the library and please send me your changes using this excellent article as guideline:

Scott Hanselman - Example: How to contribute a patch to an Open Source Project

Another place where you can found the jobs to do is in the forums. We welcome all ideas and feedback on the project.

Spread to the world !!

You can vote for the article in CodeProject so that more users know about the library. More users will send additional feedback and ideas =)

You can also post on your blog or add a link to the http://filehelpers.com web page to your web site.

Motivation

It is always good to hear what you think of the library; in the open source world we live off the user feedback and user gratitude. So, send any comments to me or post them in the forums, your comments are always welcome.
