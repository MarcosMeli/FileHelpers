FileHelpers v 2.2(Click to see the History)

FileHelpers is a free and easy to use .NET library to import or export data from fixed length or delimited records in files, strings or streams.

The idea is pretty simple:

You can strongly type your flat file (fixed or delimited) by simply describing a class that maps to each record and later read or write your file as an strongly typed .NET array

The library also has support to import or export data from different data storage types like Excel, Access, SqlServer, etc.

Basic Uses of the Library

Who needs the File Helpers Library ?

In almost every project there is a need to read or write data from/to a file of a specified format.

For example, log parsing, data warehouse and OLAP applications, communication between systems, file format transformations (for example transforming from a fixed length to a CSV file).

This library aims to provide an easy and reliable way to accomplish this task.

Main Features

Easy to use:
The FileHelpers library is straightforward to learn and use. (see Easy Example)
Auto Converters:
The library has a set of converters for common types and can be easy extended to provide custom converters. (see  ConverterBase and Converter Example)
RunTime Classes:
You can create your record class at run time, load layouts from files with source code or an XML description (see examples)
High Performance:
From version 2.0 the library doesn't use reflection to get or set the field values, it uses dynamic code generation (MSIL) and delegates
Master-Detail:
You can read and write records with a master/detail pattern. (see Example1 and Example2)
Multiple record format support:
With the MultirecordEngine, you can read files with different record layouts, you can also read files with some delimited and some fixed length records. (see Example)
Event Support:
The engines of the library contain events to make it easier to extend the behavior of the library (see Example)
Microsoft Excel Storage:
Allows you to extract and insert records between any source and an Excel file. (see ExcelDataStorage and Excel DataLink Example)
DataLink:
Provides a way to extract and insert records between a database and a file. (see DataStorage and File DataLink Example)
GenericDataLink:
Allows you to copy records between two data stores (see GenericDataLink and DataStorage)
Asynchronous Mode:
You can use the library to read line by line and not the whole file. (see Async Methods)
File Transform Engine:
To convert files in one format to another (for example a file with CSV to a fixed length record format) (see example)
Progress Notification:
Provides feedback on the progress in each operation in the library (see example)
Generics:
The cast-less and strong-typed version of the engines (see example)
Nullable Types:
The library supports Nullable types in the core (see example)
FileDiffEngine
Provides the ability to compare files with the same record layout (see example)

Others Features

Files, Stream and String Support:
You can use the library to read or write any stream or string, not only files. (see FileHelperEngine)
Different Error Behaviors:
You can set the behavior of the library when an error is found (throw and exception, ignore, save and continue, etc) (see example)
Quoted String Support
It allows you to specify that the field must be read and written as Quoted String, like Excel CSV. (see FieldQuotedAttribute)
Data Table Operations:
The library supports reading a file directly into a DataTable rather than an array
CommonEngine
An easy way to access to common operations (see example)
Encoding Aware:
You can define the encoding used to read and write files or streams. (see BaseEngine.Encoding)
Null Values:
The library can identify null values and replace these with user supplied values. (see FieldNullValueAttribute)
Good Documentation:
The library is fully documented, at least this is the intention =), and with a lot of examples of use.
Alignment and Trimming:
You can set the processing mode of the in/out strings with a lot of attributes like FieldAlignAttribute and FieldTrimAttribute.
